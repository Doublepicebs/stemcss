Theming

Stem css was built entirely on the idea of customisability.

1. Editing the theme
There are basic variables in stem:

--theme-color
--anti-theme-color

--component-background-color
--component-color

--accent-hue
--accent-saturation
--accent-lightning
--accent: hsl(var(--accent-hue), var(--accent-saturation), var(--accent-lightning))
--accent-dim: hsl(var(--accent-hue), var(--accent-saturation), calc(var(--accent-lightning) - 15%));
--accent-light: hsl(var(--accent-hue), var(--accent-saturation), calc(var(--accent-lightning) + 15%));

--text-color-on-accent


Each one of the previous variables exists once in html.light and once in html.dark. These can be tweaked to change the theme entirely.



There aren't many other variables that aren't based on color:


--border-radius


2. Making a new variant of a component

To create a new variant of a component, do the following:

A. Write the class of the variant 

button.danger {    }

B. Write the styles 
button.danger {
  background-color: var(--danger-background);
  padding: 1rem;
}

Note that --danger-background doesn't exist in stem css, so we add it in (3)




3. Creating new variables

We add a light version: 

html.light {
...
 --danger-background: #a62e2e;
...
}
And then a dark version: 
html.dark {
...
 --danger-background: #ffa6a6;
...
}
We can add only one version if it works for both dark and light: 
html {
...
 --danger-background: #a62e2e;
 ...
}




4. Changing individual components

If you want to add a red botton or a green card once, you can use html style="" to edit variables for one specific item once.


<button class="primary" style="--accent: hsl(100, 100%, 60%); --accent-dim: hsl(100, 100%, 40%);"> Hiiiii </button>

WARNING: --accent-hue, --accent-saturation AND --accent-lightning WILL NOT WORK IN THIS CASE.
